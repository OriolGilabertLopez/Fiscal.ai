Componentes del Sistema

1. Ingesta

Email (IMAP), upload web, Google Drive webhook, CSV banco

2. Extracción (OCR + LLM)

Google Vision → texto + coordenadas → GPT-4 structured output → JSON normalizado

3. Enriquecimiento

Validación NIF/IBAN, maestro contrapartes, clasificación contable automática

4. Motor Fiscal

Reglas declarativas YAML por régimen, cálculo IVA/IRPF, calendario obligaciones

5. Conciliación

Matching documento ↔ movimiento (importe, fecha ±3d, fuzzy text)

6. HITL & Reporting

Cola revisión, explicabilidad, generación Excel/PDF, auditoría

Pipeline de Procesamiento

1

Upload & Validación

Usuario sube PDF → Validar formato/tamaño → Generar hash SHA-256 → Guardar en S3

⏱️ ~1 segundo
2

OCR

Google Vision API extrae texto + coordenadas → Layout heuristics (detecta bloques)

⏱️ ~2-3 segundos
3

Extracción IA

GPT-4 Turbo con prompt estructurado → JSON (proveedor, NIF, fecha, base, IVA, total, confidence)

⏱️ ~3-5 segundos
4

Validación de Negocio

Total = Base + IVA (±0.02€) • Fecha válida • Tipo IVA ∈ {21, 10, 4}% → Si confidence < 0.85 → HITL

⏱️ <1 segundo
5

Clasificación & Almacenamiento

Emitida→IVA devengado / Recibida→IVA soportado → Upsert contrapartes → PostgreSQL

⏱️ <1 segundo
6

Cálculo Fiscal & Export

Motor fiscal aplica reglas → Genera Excel/PDF con Modelo 303 → Auditoría completa

⏱️ ~1 segundo

Tiempo total de procesamiento: ~10-15 segundos por factura

Objetivo: 98% precisión en campos clave (NIF, fecha, totales)

Riesgos & Mitigación

OCR/LLM fallan en tickets arrugados/mal escaneados

Probabilidad: Alta | Impacto: Medio

Mitigación: HITL obligatorio + guía de captura móvil + dataset sintético robusto + fallback a corrección manual en 1 click

Cambios normativos fiscales

Probabilidad: Media | Impacto: Alto

Mitigación: Motor de reglas YAML versionado + tests unitarios por modelo (303/130) + suscripción BOE + asesor fiscal en retainer

Privacidad / brecha de datos

Probabilidad: Baja | Impacto: Crítico

Mitigación: Cifrado end-to-end + auditorías security trimest + penetration testing + seguro ciberseguridad + opción on-prem para gestorías

Lock-in de proveedor LLM

Probabilidad: Media | Impacto: Medio

Mitigación: Abstracción de proveedor (adapter pattern) + tests regresión por campos + dataset golden standard + eval pipeline automatizado
